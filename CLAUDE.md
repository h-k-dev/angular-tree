# angular-tree

High-performance standalone Angular tree component (react-arborist internals, Angular Material DX). Angular ≥ 21, zoneless, signals-only, CDK as the only runtime dependency.

## Session protocol — do this first, every session

1. Read **ROADMAP.md** — the single source of truth for phases, locked decisions, settled brainstorming, and open questions. Resume work from the current phase noted there. Post-v1 planning lives in **ROADMAP2.md** (nothing there starts before Phase 8 completes).
2. Read **STYLE.md** — mandatory coding style. All code (including stubs and specs) must pass its Review Checklist.

## Rules

- **Locked decisions in ROADMAP.md are not renegotiated silently.** If implementation reveals a problem with one, raise it explicitly before deviating.
- **When a decision gets settled in discussion, write it into ROADMAP.md immediately** (settled list, decisions table, or non-goals) — the roadmap must never lag the conversation.
- Public API changes go through the "Public API Sketch" section of ROADMAP.md before code.
- DX and Material familiarity outrank micro-optimization at the API boundary; performance work stays internal (flat model, virtualization).
- The tree ships no UI it doesn't own: no built-in menu, editor, or checkbox — directives + template contracts instead.
- **Scaffold with the Angular CLI** (`ng generate service|directive|component …`), never by hand — current Angular defaults apply (no `.service.ts`/`.directive.ts` file suffixes).
- **Modern Angular naming: no `Service`/`Directive`/`Component` class suffixes.** Names describe what the thing _does_ (`TreeController`, `TreeNodeToggle`, `AngularTree`) — see STYLE.md § Naming.

A "Lazy Load Only" example added 2026-07-11 (`lazy-load-example`, route `/lazy`) — five self-contained roots, each backed by a DIFFERENT public API (a GitHub repo via the contents API, a GBIF taxonomy, a Hacker News thread, a Wikipedia category, and an npm package's files via jsDelivr's data API — unpkg's `?meta` redirect drops CORS so it can't be fetched from the browser), all unified behind ONE tree by the accessor contract (`lazy-sources.ts` maps each response to a common `LazyNode`; the tree never learns the five shapes). Children fetch ONE LEVEL AT A TIME on open (jsDelivr is the deliberate exception — no per-dir endpoint, so its whole tree arrives in one request and deeper folders open instantly: lazy render, not lazy fetch). The accessor returns a COLD Observable (`defer`), NEVER a Promise — the tree probes accessors at flatten time for every loaded node to learn expandability, and a Promise-returning accessor starts its fetch at probe time; combined with the write-back this self-crawled all five backends on page load (~10k requests, caught 2026-07-11 same day). Only expand-intent subscribes (`ensureChildren` → `firstValueFrom`), so probing is free: page load = 0 API requests, one expand = exactly one (regression spec pins the cold probe; the footgun is now documented on `TreeChildrenAccessor`'s JSDoc). Drag & drop is sandboxed per `source` via `disableDrop` (no dragging across data sources; also forbids root drops and drops into an unfetched branch). Because the data is truly lazy the component OWNS a progressively-loaded `roots` signal — fetched children are written back with an IDENTITY-PRESERVING attach (only the loaded node + its ancestors get new objects, so the controller's WeakMap-per-node accessor memoisation survives and sibling branches are never re-fetched by the write), which is what lets the controlled `moved` intent apply through `applyMove`. Per-source error isolation is visible in the demo: an exhausted GitHub rate limit shows that one root's Retry while the other four load. The Resource (HTML/TS/SCSS/Data), Static (SCSS/Data), VS Code (HTML/TS/SCSS/Data) and Lazy Load Only (HTML/TS/SCSS/Data) examples all carry PrimeNG-style Shiki source tabs 2026-07-11 — real files copied to `source/` assets (angular.json) and highlighted lazily, zero drift.
